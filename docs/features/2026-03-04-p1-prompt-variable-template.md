# FEAT-015：P1 提示词变量模板（{{变量}}）

## 1. 元信息
- 功能 ID：FEAT-015
- 功能名称：实现提示词变量模板填写与一键插入
- 日期：2026-03-04
- 作者：Codex
- 分支：N/A（工作区直接更新）

## 2. 概述
继续推进 P1，在提示词库中支持变量模板能力。提示词正文中使用 `{{变量名}}` 占位后，可在卡片中展开变量面板填写参数，再“填充并插入”到 ChatGPT 输入框。

## 3. 需求映射
- 对应 PRD 章节：6.2（P1：提示词分组与变量模板）、8（FR-05 提示词库）
- 覆盖的用户场景：
  - 用户希望维护一个模板，按不同任务填不同变量后复用。
  - 用户不希望每次手动编辑模板正文。
- 本次范围：
  - 自动识别 `{{变量}}`
  - 变量输入面板（按提示词卡片展开）
  - 校验未填写变量并提示
  - 填充后插入到输入框
- 本次不包含：
  - 变量类型（日期/下拉/必选）
  - 变量默认值持久化策略增强

## 4. 设计
- 关键设计决策：
  - 变量解析规则：`{{\s*name\s*}}`，允许 `a-zA-Z0-9_.-`。
  - 未填写变量时阻止插入，并给出“仍有 N 个变量未填写”提示。
  - 支持“清空变量值”快速重置当前模板参数。
- 取舍说明：
  - 先做轻量文本变量替换，后续再补高级变量控件。
- 权限/数据/隐私考虑：
  - 变量填写在前端本地状态中处理，无新增权限和外发数据。

## 6. 验证
- 自动化测试：
  - `npm run typecheck`
  - `npm run build`
- 手工验证步骤：
  1. 新建提示词：`请写一篇关于 {{topic}} 的文章，风格 {{tone}}`。
  2. 在卡片点击“变量(2)”并填写 `topic/tone`。
  3. 点击“填充并插入”，确认插入内容变量已替换。
  4. 任意变量留空时点击插入，确认出现未填写提示。
- 测试结果摘要：
  - 类型检查与构建通过，变量填充链路可用。
