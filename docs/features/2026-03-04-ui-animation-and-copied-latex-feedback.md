# FEAT-021：动效优化与已复制 LaTeX 可视反馈

## 1. 元信息
- 功能 ID：FEAT-021
- 功能名称：优化交互动效并展示最近复制的 LaTeX
- 日期：2026-03-04
- 作者：Codex
- 分支：N/A（工作区直接更新）

## 2. 概述
根据用户反馈“动画更舒服，且要看到已复制的 LaTeX”，对侧边栏交互动效做了柔化调优，并在公式工作台增加“最近复制的 LaTeX”可视区块，明确复制结果。

## 3. 需求映射
- 对应 PRD 章节：9（体验与交互质量）
- 覆盖的用户场景：
  - 用户需要更顺滑的交互反馈，不希望突兀。
  - 用户复制公式后需要确认复制内容。
- 本次范围：
  - 面板/按钮/卡片过渡动画优化
  - 状态提示淡入动画
  - 公式复制结果展示（来源 + TeX 内容）
- 本次不包含：
  - 全局动画配置中心
  - 复杂时间线动画编排

## 4. 设计
- 关键设计决策：
  - 统一使用较柔和的 cubic-bezier 与轻微位移动效。
  - 复制结果在公式工作台内就地展示，避免用户视线跳转。
  - 复制来源（页面点击/工作台）与 LaTeX 原文同时显示。
- 取舍说明：
  - 控制动效强度，避免“花哨感”和性能负担。
- 权限/数据/隐私考虑：
  - 纯前端交互优化，无新增权限。

## 5. 实现
- 主要变更文件：
  - `src/content/App.tsx`
  - `src/content/index.tsx`
- 核心逻辑：
  - `App.tsx`

  - `index.tsx`
    - 新增/优化：`gv-soft-rise`、`gv-fade-in`、卡片 hover 过渡、按钮过渡
    - 新增复制结果样式：`.gv-formula-copied`
- 数据模型/存储变更：
  - 无
- 配置/Manifest 变更：
  - 无

## 6. 验证
- 自动化测试：

- 手工验证步骤：
  1. 点击页面公式或在公式工作台点击“复制 TeX”。
  2. 确认出现“已复制 LaTeX”状态与最近复制内容。

## 7. 风险与回滚
- 已知风险：
  - 低性能设备上动效过多可能轻微影响流畅度。
- 回滚策略：
  - 回退 `index.tsx` 动效样式和 `App.tsx` 复制反馈区块。
