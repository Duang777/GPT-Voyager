# FR-06：单会话 Markdown 导出

## 1. 元信息
- 功能 ID：FEAT-005
- 功能名称：实现当前会话一键导出 Markdown
- 日期：2026-03-04
- 作者：Codex
- 分支：N/A（工作区直接更新）

## 2. 概述
在侧边栏“会话索引”区域新增“导出会话”按钮。用户可将当前会话内容提取为 Markdown 并下载到本地文件，导出内容包含会话标题、导出时间、会话链接及按角色分段的消息正文。

## 3. 需求映射
- 对应 PRD 章节：6.1（P0）、8（FR-06）、10（技术方案）、15（DoD）
- 覆盖的用户场景：
  - 用户希望把当前对话快速归档为本地 Markdown 文件。
  - 用户希望导出内容具备结构化可读性（角色分段）。
- 本次范围：
  - 提取当前会话消息
  - 生成 Markdown 文本
  - 触发本地下载
  - 侧边栏状态反馈（成功/失败）
- 本次不包含：
  - 批量会话导出
  - 自定义导出模板
  - PDF/HTML 导出

## 4. 设计
- 关键设计决策：
  - 通过 DOM 属性 `[data-message-author-role]` 抽取会话消息。
  - 仅在 `/c/<id>` 会话页面允许导出，避免在非会话页误操作。
  - 文件名基于会话标题，自动清洗非法字符。
  - 使用 `Blob + objectURL` 触发浏览器下载。
- 取舍说明：
  - 当前导出聚焦纯文本可读性，不保留复杂富文本样式。
  - 对 DOM 结构变化敏感，后续可增加多策略解析。
- 权限/数据/隐私考虑：
  - 导出文件完全本地生成与下载，不上传任何内容。
  - 无新增权限需求。

## 5. 实现
- 主要变更文件：
  - `src/content/conversationExport.ts`
  - `src/content/App.tsx`
  - `src/content/index.tsx`
  - `README.md`
- 核心逻辑：
  - `conversationExport.ts`：
    - `collectConversationMessages`：提取消息角色与文本。
    - `buildConversationMarkdown`：组装 Markdown 结构。
    - `exportCurrentConversationToMarkdown`：校验页面、生成文件、触发下载。
  - `App.tsx`：
    - 会话索引区增加“导出会话”按钮与状态提示。
    - 导出结果统一反馈“已导出/失败原因”。
  - `index.tsx`：
    - 新增导出状态文本样式与动作按钮组样式。
- 数据模型/存储变更：
  - 无新增持久化键。
- 配置/Manifest 变更：
  - 无。

## 6. 验证
- 自动化测试：
  - `npm run typecheck`
  - `npm run build`
- 手工验证步骤：
  1. 打开任一 `/c/<id>` 会话页面
  2. 点击“导出会话”
  3. 检查是否下载 `.md` 文件
  4. 打开文件验证标题、时间、链接与消息分段
  5. 在非会话页面点击导出，确认提示失败原因
- 测试结果摘要：
  - 类型检查和构建通过，导出流程可执行。

## 7. 风险与回滚
- 已知风险：
  - DOM 结构变化可能导致消息提取失败。
  - 极长会话导出时文件体积较大。
- 回滚策略：
  - 回退 `conversationExport.ts` 与 `App.tsx` 到 FEAT-004 版本。
