# FEAT-048：去绿色化高亮重构（中性灰黑强调）

## 1. 元信息
- 功能 ID：FEAT-048
- 功能名称：去绿色化高亮重构（中性灰黑强调）
- 日期：2026-03-05
- 作者：Codex
- 分支：main

## 2. 概述
根据反馈“绿色高亮很丑”，本次将面板内所有主要绿色高亮改为中性灰黑体系，保留极简白色风格并提升视觉克制感。

## 3. 需求映射
- 需求：
  - 去除绿色高亮框与绿色强调底色
  - 维持清晰的状态区分与可读性
- 场景：
  - 导航选中、会话卡片激活、按钮高亮、焦点态、时间线定位、公式复制反馈

## 4. 实现
- 主要变更文件：
  - `src/content/index.tsx`
  - `src/content/App.tsx`
  - `docs/FEATURE_LOG.md`

### 4.1 设计令牌调整
- `--gv-accent`：由绿色改为石墨灰 `#374151`
- `--gv-accent-soft`：改为浅灰 `#f3f4f6`
- `--gv-focus`：改为中性灰蓝焦点环 `rgba(51, 65, 85, 0.24)`

### 4.2 状态样式去绿
- 导航选中态、按钮 active、卡片 active、标签 active、星标 active 统一到灰蓝边框与浅灰底。
- `section-highlight` 与统计 CTA 的边框/背景从绿色系改为灰蓝系。

### 4.3 交互反馈去绿
- 页面公式复制提示（Word）改为中性灰提示色。
- 时间线高亮 outline 从绿色改为灰蓝色。
- 公式/会话定位 outline 从绿色改为 `#64748b`。

## 5. 验证
- 自动化验证：
  - `npm run typecheck` 通过
  - `npm run build` 通过
- 手工验证：
  1. 选中“会话工作台”导航，边框与底色为灰蓝，不再偏绿。
  2. 星标区 CTA、会话激活卡片、标签激活均为中性高亮。
  3. 点击公式复制反馈与时间线定位描边不再出现绿色。

## 6. 风险与回滚
- 已知风险：
  - 去绿后品牌识别度略下降，但整体更克制。
- 回滚策略：
  - 回退 `src/content/index.tsx` 与 `src/content/App.tsx` 的本次配色提交。
