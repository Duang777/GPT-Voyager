# 功能记录

## 1. 元信息
- 功能 ID：FIX-018
- 功能名称：修复下拉框再次点击无法收起
- 日期：2026-03-05
- 作者：Codex
- 分支：main

## 2. 概述
- 修复自定义下拉在部分场景下“再次点击触发器无法收起”的交互问题。  
- 核心是统一事件边界：组件内点击不触发外部关闭，触发器点击强制走显式开关逻辑。

## 3. 需求映射
- 对应 PRD 章节：`FR-01 侧边栏`
- 覆盖的用户场景：
  - 用户点击同一下拉触发器可稳定开/关菜单。
- 范围说明（In scope / Out of scope）：
  - In scope：下拉开关事件处理。
  - Out of scope：下拉视觉样式。

## 4. 设计
- 关键设计决策：
  - 外部点击关闭从“触发器/菜单双判断”改为“组件根容器判断”。
  - 触发器与选项点击增加 `preventDefault + stopPropagation`，避免冒泡链导致状态竞争。

## 5. 实现
- 主要变更文件：
  - `src/content/App.tsx`
- 核心逻辑：
  - 调整 `VoyagerSelect` 的 `onPointerDown` 外部关闭判断逻辑。
  - 触发器 `onClick` 与选项 `onClick` 添加事件抑制并显式切换状态。

## 6. 验证
- 自动化测试：
  - `npm run typecheck` 通过。
  - `npm run build` 通过。
- 手工验证步骤：
  1. 连续点击同一下拉触发器，验证可开可关。
  2. 点击下拉外部区域，验证可关闭。
  3. 点击选项，验证选中并关闭。
- 测试结果摘要：
  - 下拉开关行为恢复稳定。
