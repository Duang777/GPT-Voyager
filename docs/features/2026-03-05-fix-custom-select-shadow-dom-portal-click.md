# 功能记录

## 1. 元信息
- 功能 ID：FIX-016
- 功能名称：修复自定义下拉在 Shadow DOM 下无法正常展开点击
- 日期：2026-03-05
- 作者：Codex
- 分支：main

## 2. 概述
- 修复“下拉框点不出来”的问题。  
- 原因是自定义下拉菜单通过 portal 挂载到 `document.body`，而样式定义在扩展 Shadow DOM 内，导致菜单层样式与交互异常。

## 3. 需求映射
- 对应 PRD 章节：`FR-01 侧边栏`
- 覆盖的用户场景：
  - 用户在各页面点击下拉后可正常打开和选择选项。
- 范围说明（In scope / Out of scope）：
  - In scope：下拉菜单渲染容器修复。
  - Out of scope：下拉外观设计本身。

## 4. 设计
- 关键设计决策：
  - 取消将菜单 portal 到 `document.body`。
  - 改为在组件内直接渲染菜单层，保持在 Shadow DOM 样式作用域内。
- 取舍说明：
  - 保持样式一致与可交互稳定性优先。

## 5. 实现
- 主要变更文件：
  - `src/content/App.tsx`
- 核心逻辑：
  - 移除 `createPortal` 引入与调用。
  - 下拉菜单改为组件内条件渲染。
- 数据模型/存储变更：
  - 无。

## 6. 验证
- 自动化测试：
  - `npm run typecheck` 通过。
  - `npm run build` 通过。
- 手工验证步骤：
  1. 在会话索引、时间线、设置页点击任意下拉。
  2. 确认菜单能展开并可点击切换。
- 测试结果摘要：
  - 下拉恢复可点击，交互正常。
