# FEAT-016：前端对齐与简约化重构（UI polish）

## 1. 元信息
- 功能 ID：FEAT-016
- 功能名称：统一控件尺寸、网格与间距，修复“框未对齐”问题
- 日期：2026-03-04
- 作者：Codex
- 分支：N/A（工作区直接更新）

## 2. 概述
根据用户反馈“前端不够简约且框没对齐”，本次对界面做了一轮结构化对齐优化：统一输入/按钮/下拉高度，收敛区块间距，修正统计区与筛选区布局，并将提示词卡片改为更规整的垂直信息流。

## 3. 需求映射
- 对应 PRD 章节：9（可维护性/体验）、15（DoD 可用性）
- 覆盖的用户场景：
  - 用户长时间使用侧边栏，希望视觉规整、操作负担低。
- 本次范围：
  - 控件高度统一
  - 统计区布局优化
  - 筛选区对齐优化
  - 提示词卡片与变量面板布局优化
- 本次不包含：
  - 主题切换系统
  - 全新视觉语言重设计

## 4. 设计
- 关键设计决策：
  - 统一基线高度：按钮/输入/下拉最小高度统一到 34px（头部按钮 32px）。
  - 统计区采用“2x2 数据卡 + 单独 CTA”结构，避免 5 卡不齐。
  - 筛选区补充 `compact` 变体，减少提示词页控件拥挤。
- 取舍说明：
  - 优先可读性和一致性，弱化装饰和复杂层次。
- 权限/数据/隐私考虑：
  - 纯样式与结构优化，不涉及权限和数据外发。
