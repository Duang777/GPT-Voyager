# 功能记录

## 1. 元信息
- 功能 ID：FEAT-051 / FIX-011
- 功能名称：分类卡片化进入 + 会话页顶部间距修复
- 日期：2026-03-05
- 作者：Codex
- 分支：main

## 2. 概述
- 处理两项用户反馈：  
- 会话页顶部“工作台说明卡片”和下方模块区域存在贴边/重叠感。  
- 分类管理应改为“一个分类一个卡片，并支持直接进入”。  
- 本次将分类模块升级为卡片导航，并优化顶部区域间距与结构显示。

## 3. 需求映射
- 对应 PRD 章节：`FR-01 侧边栏`、`FR-03 分类管理`
- 覆盖的用户场景：
  - 用户在会话页浏览时，顶部卡片区与模块区不再视觉冲突。
  - 用户在分类管理中可直接按分类卡片进入对应会话列表。
- 范围说明（In scope / Out of scope）：
  - In scope：会话页顶部结构显示策略、分类管理模块布局与进入动作。
  - Out of scope：分类规则自动化、拖拽排序。

## 4. 设计
- 关键设计决策：
  - 在会话视图隐藏顶部 `view intro` 重复说明卡，仅保留工作台入口。
  - 顶部 sticky 区增加底部间距，降低贴边重叠观感。
  - 分类模块改为卡片网格：全部会话 / 未分类 / 各文件夹卡片。
  - 每个分类卡片提供“进入”动作，进入后自动切换到会话索引视图并应用筛选。
- 取舍说明：
  - 分类区信息更结构化，但相较原 chip 列表占用稍大高度。
- 权限/数据/隐私考虑：
  - 纯前端布局与交互优化，不涉及新增权限与数据结构变更。

## 5. 实现
- 主要变更文件：
  - `src/content/App.tsx`
  - `src/content/index.tsx`
  - `PRD.md`
- 核心逻辑：
  - 新增 `enterFolderFromClassification`，复用 `openFolderView` 并切换到 `index` 子工作台。
  - 分类管理新增 `gv-folder-card-grid` 卡片列表，按分类展示计数与操作。
  - 会话页顶部在 `activeView === "conversations"` 时不再渲染 `gv-view-intro`。
  - 调整 `gv-topbar` 底部外边距与内边距，缓解视觉重叠。
- 数据模型/存储变更：
  - 无。
- 配置/Manifest 变更：
  - 无。

## 6. 验证
- 自动化测试：
  - `npm run typecheck` 通过。
  - `npm run build` 通过。
- 手工验证步骤：
  1. 打开会话工作台，确认顶部不再重复出现说明卡片叠压感。
  2. 进入“分类管理”，确认按卡片展示分类。
  3. 点击任一卡片“进入”，确认跳转到会话索引并应用对应分类筛选。
- 测试结果摘要：
  - 视觉间距问题得到缓解，分类进入链路可用。

## 7. 风险与回滚
- 已知风险：
  - 分类数量极多时卡片区高度增长明显。
- 回滚策略：
  - 回滚 `App.tsx` 分类区卡片结构和顶部显示条件；
  - 回滚 `index.tsx` 卡片与 topbar 间距样式改动。
