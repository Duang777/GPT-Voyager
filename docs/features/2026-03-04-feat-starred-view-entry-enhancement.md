# FEAT-013：星标会话入口增强（可点击总览 + 独立星标区）

## 1. 元信息
- 功能 ID：FEAT-013
- 功能名称：增强星标会话可用性与入口可发现性
- 日期：2026-03-04
- 作者：Codex
- 分支：N/A（工作区直接更新）

## 2. 概述
针对“星标后不知道怎么集中查看”的反馈，补充了两个明确入口：
1) 统计区“星标会话”卡片可点击，直接进入/退出仅星标视图；
2) 新增“星标会话”独立列表区，可一键打开星标会话并查看备注。

## 3. 需求映射
- 对应 PRD 章节：6.2（P1 备注与元数据效率）、9（可用性）
- 覆盖的用户场景：
  - 用户希望打星后能快速聚合查看，不需要再手动找筛选按钮。
- 本次范围：
  - 星标统计卡交互化
  - 独立星标区展示（最多 8 条）
  - 空状态提示优化
- 本次不包含：
  - 星标分页与排序自定义

## 4. 设计
- 关键设计决策：
  - 保持原有“仅星标”按钮，同时新增更直观入口，避免隐藏功能。
  - 星标区展示标题、时间、备注摘要，减少跳转前信息不确定性。
- 取舍说明：
  - 先做轻量列表，不引入额外复杂状态和分页。
- 权限/数据/隐私考虑：
  - 仅 UI 与本地状态逻辑调整，不涉及新增权限。

## 5. 实现
- 主要变更文件：
  - `src/content/App.tsx`
  - `src/content/index.tsx`
- 核心逻辑：
  - 新增 `activateStarredOnly` 入口逻辑。
  - 星标统计卡改为按钮，点击进入/退出仅星标。
  - 新增 `starredConversations` 派生列表并渲染独立区块。
  - 在“仅星标”筛选下优化空结果提示。
- 数据模型/存储变更：
  - 无
- 配置/Manifest 变更：
  - 无

## 6. 验证
- 自动化测试：
  - `npm run typecheck`
  - `npm run build`
- 手工验证步骤：
  1. 给会话打星，点击统计卡“星标会话”，确认进入仅星标视图。
  2. 在“星标会话”独立区点击条目，确认可直接打开目标会话。
  3. 退出仅星标后，确认返回全部会话视图。
  4. 在无星标数据时，确认有明确引导提示。
- 测试结果摘要：
  - 类型检查和构建通过，入口可用性明显提升。

## 7. 风险与回滚
- 已知风险：
  - 统计卡可点击可能与纯展示认知有差异，需要文案引导。
- 回滚策略：
  - 回退 `App.tsx` 星标入口增强与 `index.tsx` 对应样式。
