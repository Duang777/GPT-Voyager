# FEAT-009：白色/米白简约主题升级

## 1. 元信息
- 功能 ID：FEAT-009
- 功能名称：侧边栏视觉重构为白色/米白简约风格
- 日期：2026-03-04
- 作者：Codex
- 分支：N/A（工作区直接更新）

## 2. 概述
根据用户反馈“界面仍然偏丑、希望更接近 GPT 的简约舒缓风格”，将侧边栏主题从深色渐变方案改为白色/米白基础色系，保留信息结构不变，重点优化可读性、层级和操作反馈。

## 3. 需求映射
- 对应 PRD 章节：6.1（P0 可用性）、9（可维护性/交互体验）
- 覆盖的用户场景：
  - 用户长时间在 ChatGPT 页面进行整理，希望界面轻量、低视觉噪声。
  - 用户需要快速识别当前分区和激活状态。
- 本次范围：
  - 统一组件配色、边框、阴影、状态色
  - 导航、按钮、列表项、表单控件的视觉一致性收敛
- 本次不包含：
  - 功能结构调整
  - 新增复杂动效

## 4. 设计
- 关键设计决策：
  - 主背景使用 `#faf9f6` / `#f6f4ef` / `#ffffff` 的浅色层次。
  - 交互强调色使用近 GPT 风格的绿色系 `#10a37f`。
  - 卡片与列表采用低对比边框 + 低强度阴影，减少“花哨感”。
- 取舍说明：
  - 弱化动画，优先稳定和阅读舒适性。
- 权限/数据/隐私考虑：
  - 纯前端样式调整，不涉及权限与数据流变化。

## 5. 实现
- 主要变更文件：
  - `src/content/index.tsx`
- 核心逻辑：
  - 在 Shadow DOM 样式中重定义主题变量：背景色、文本色、分割线、强调色。
  - 调整按钮/标签/卡片在 hover 与 active 的表现。
  - 新增隐藏文件输入样式类（服务备份导入按钮绑定）。
- 数据模型/存储变更：
  - 无
- 配置/Manifest 变更：
  - 无

## 6. 验证
- 自动化测试：
  - `npm run typecheck`
  - `npm run build`
- 手工验证步骤：
  1. 加载扩展后打开 ChatGPT 页面，确认侧边栏为白色/米白整体风格。
  2. 切换三个分区，观察导航激活态与卡片层级是否清晰。
  3. 测试按钮 hover/focus 和输入框 focus 样式是否正常。
- 测试结果摘要：
  - 类型检查和构建通过，视觉样式生效。
