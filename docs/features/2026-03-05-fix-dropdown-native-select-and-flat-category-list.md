# 功能记录

## 1. 元信息
- 功能 ID：FIX-012 / UI-012
- 功能名称：下拉展开改为原生选择器 + 分类卡片扁平单列化
- 日期：2026-03-05
- 作者：Codex
- 分支：main

## 2. 概述
- 处理“所有这种展开都很丑”的反馈：移除自定义下拉浮层（`InlineSelect`），统一改为原生 `select` 下拉，避免展开层遮挡/错位/边框突兀。  
- 同时将分类卡片从多列块状改为更轻量的单列扁平列表，保持“一分类一卡片，可进入”。

## 3. 需求映射
- 对应 PRD 章节：`FR-01 侧边栏`、`FR-03 分类管理`
- 覆盖的用户场景：
  - 时间线/公式筛选展开时，视觉不再“浮层突兀”。
  - 分类管理更接近信息流样式，阅读与操作更顺手。
- 范围说明（In scope / Out of scope）：
  - In scope：下拉控件实现方式、筛选视觉样式、分类卡片布局样式。
  - Out of scope：筛选逻辑和分类数据模型。

## 4. 设计
- 关键设计决策：
  - 删除自定义 `InlineSelect` 组件，减少展开层 UI 不一致问题。
  - 采用统一 `gv-select` 原生下拉样式（轻边框、简洁箭头、更柔和焦点态）。
  - 分类列表改为单列扁平卡片，降低块状堆叠感。
- 取舍说明：
  - 放弃自定义下拉动画和高度可控菜单，换取稳定、系统一致、视觉更克制的体验。
- 权限/数据/隐私考虑：
  - 纯前端样式与组件结构变更，不涉及权限和数据。

## 5. 实现
- 主要变更文件：
  - `src/content/App.tsx`
  - `src/content/index.tsx`
- 核心逻辑：
  - 删除 `InlineSelect` 组件定义。
  - 时间线角色筛选、公式类型筛选改为原生 `select`。
  - 删除 `gv-inline-select*` 样式，统一 `gv-select` 视觉。
  - 分类卡片样式改为单列扁平布局，并在小屏下保持同一风格。
- 数据模型/存储变更：
  - 无。
- 配置/Manifest 变更：
  - 无。

## 6. 验证
- 自动化测试：
  - `npm run typecheck` 通过。
  - `npm run build` 通过。
- 手工验证步骤：
  1. 打开会话时间线、公式工作台，展开筛选下拉。
  2. 检查下拉是否为系统原生样式，不再出现自定义蓝色浮层错位。
  3. 打开分类管理，检查分类为单列扁平卡片并可“进入”。
- 测试结果摘要：
  - 下拉展开视觉明显简化；分类区更接近轻量列表风格。

## 7. 风险与回滚
- 已知风险：
  - 原生下拉在不同操作系统下外观会有轻微差异。
- 回滚策略：
  - 回滚 `App.tsx` 的 `select` 替换以及 `index.tsx` 的样式重构。
