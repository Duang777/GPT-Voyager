# FEAT-024：会话索引简约筛选 + 文件夹快速打开

- 功能 ID：FEAT-024

- 日期：2026-03-04
- 作者：Codex
- 分支：N/A

## 2. 概述
针对“文件夹分类后无法直观看到对应文件夹内容、筛选区信息密度过高导致困惑”的反馈，本次迭代重构会话索引交互：新增“文件夹快速打开”按钮组，支持一键切换到目标文件夹/未分类；将筛选默认模式简化为常用操作，高级筛选改为可折叠，批量面板默认收起以降低视觉复杂度。

## 3. 需求映射
- 对应 PRD 章节：
  - 8（FR-03 分类管理、FR-04 搜索）
  - 9（可维护性与可用性交互体验）
- 覆盖的用户场景：
  - 用户将会话分配到文件夹后，希望能直接“打开这个文件夹”查看对应会话。
  - 用户在高频筛选场景下希望界面更简洁，不被批量和高级条件打断。
- 范围说明（In scope / Out of scope）：
  - In scope：
    - 文件夹快速打开条（全部/未分类/各文件夹，含计数）
    - 默认简约筛选区（仅星标 + 高级筛选折叠 + 清空）
    - 批量区默认收起，按需展开
    - 会话卡片内增加“打开该文件夹”按钮
  - Out of scope：
    - 文件夹树形层级
    - 跨页面持久化“筛选面板展开状态”
    - 批量操作流程重做

## 4. 设计
- 关键设计决策：
  - 文件夹筛选以“按钮组 + 计数”替代单一下拉入口，强化可发现性与可点击性。
  - 高级筛选（文件夹下拉/标签下拉）默认隐藏，只在需要时展开。
  - 批量操作作为低频动作默认收起，保留快捷键和撤销能力。
- 取舍说明：
  - 保留高级筛选下拉，避免一次性移除导致功能回归。
  - 未新增复杂动画，优先保证结构清晰和信息层级。
- 权限/数据/隐私考虑：
  - 无新增权限，无网络传输，数据继续本地存储。

## 6. 验证
- 自动化测试：
  - `npm run typecheck`：通过
  - `npm run build`：通过
- 手工验证步骤：
  1. 新建并分配文件夹后，在“打开文件夹”区域点击目标文件夹，确认列表切换。
  2. 点击“未分类”按钮，确认仅显示未分配文件夹的会话。
  3. 切换“高级筛选”展开/收起，验证标签与文件夹下拉可用。
  4. 未勾选会话时观察批量区默认收起，勾选后可展开执行批量操作。
  5. 在会话卡片点击“打开该文件夹”，确认视图切换到对应文件夹。
- 测试结果摘要：
  - 类型检查和构建通过，简约筛选与文件夹直达链路可用。

## 7. 风险与回滚
- 已知风险：
  - 部分用户可能习惯原筛选布局，初次会有位置变更适应成本。
  - 文件夹计数基于本地索引，若索引未刷新可能与页面即时状态有短时差。
- 回滚策略：
  - 回退 `App.tsx` 会话索引筛选与批量面板结构。
  - 回退 `index.tsx` 新增样式块。
  - 回退 `README.md` 与 `PRD.md` 对应描述。
