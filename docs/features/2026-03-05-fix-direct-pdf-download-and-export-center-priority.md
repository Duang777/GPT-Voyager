# 功能记录

## 1. 元信息
- 功能 ID：FIX-006 / UX-006
- 功能名称：PDF 直接下载 + 导出中心前置
- 日期：2026-03-05
- 作者：Codex
- 分支：main

## 2. 概述
- 将“导出 PDF”从“打开网页再打印”改为“直接生成并下载 `.pdf` 文件”，避免用户点击后只看到网页的体验问题；同时把会话导出入口前置到“会话工作台”顶部，降低操作路径并提升可发现性。

## 3. 需求映射
- 对应 PRD 章节：`6.2 P1`、`FR-06 导出/导入`
- 覆盖的用户场景：
  - 用户点击“导出 PDF”后希望直接拿到 PDF 文件，而不是中间网页。
  - 用户希望导出入口在面板最前面，减少查找成本。
- 范围说明（In scope / Out of scope）：
  - In scope：会话 PDF 导出链路、导出按钮布局顺序、导出状态提示文案。
  - Out of scope：PDF 排版高级模板、分页智能断点、批量会话 PDF 导出。

## 4. 设计
- 关键设计决策：
  - 使用 `html2canvas + jsPDF` 在内容脚本中直接生成 PDF 并下载。
  - PDF 生成基于离屏渲染容器，避免影响页面当前可见布局。
  - 导出入口独立为“导出中心”区块，置于会话工作台靠前位置。
- 取舍说明：
  - 当前采用整页截图分页写入 PDF，保证兼容和实现稳定；代价是超长会话导出内存占用较高。
- 权限/数据/隐私考虑：
  - 全程本地渲染与本地下载，不新增远端请求与权限。

## 6. 验证
- 自动化测试：
  - `npm run typecheck` 通过。
  - `npm run build` 通过。
- 手工验证步骤：
  1. 打开任意 ChatGPT 会话详情页（`/c/...`）。
  2. 打开 GPT Voyager 面板，确认“导出中心”位于会话页前部。
  3. 点击“导出 PDF”，观察浏览器直接下载 `.pdf` 文件。
  4. 打开导出的 PDF，确认包含文本和图片内容。
- 测试结果摘要：
  - 编译通过；导出入口位置符合预期；PDF 导出不再打开中间网页。

## 7. 风险与回滚
- 已知风险：
  - 极长会话导出时，整页渲染方式可能导致性能下降或耗时增加。
  - 少量跨域图片可能因 CORS 限制无法完整绘制。
- 回滚策略：
  - 回滚 `src/content/conversationExport.ts` 到上一版“新窗口打印”实现；
  - 回滚 `App.tsx` 导出区布局调整。
