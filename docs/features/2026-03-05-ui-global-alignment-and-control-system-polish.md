# 功能记录

## 1. 元信息
- 功能 ID：UI-017
- 功能名称：全局控件系统与对齐规则优化（按钮/输入/下拉/面板）
- 日期：2026-03-05
- 作者：Codex
- 分支：main

## 2. 概述
- 针对“按钮丑、排版不规则、对齐混乱”的反馈，进行一轮全局 UI 规范化。  
- 统一控件尺寸、圆角、按钮最小宽度和标题区按钮布局；同时降低过度 hover 动效，改成更克制的简约风格。

## 3. 需求映射
- 对应 PRD 章节：`FR-01 侧边栏`
- 覆盖的用户场景：
  - 用户在会话、提示词、设置等页面看到一致的控件尺寸和对齐节奏。
  - 按钮与下拉在窄面板中不再出现随机宽度和错位。
- 范围说明（In scope / Out of scope）：
  - In scope：全局样式基线、提示词页头部操作区布局、会话项控制区按钮对齐、下拉菜单锚点规则。
  - Out of scope：业务逻辑与数据结构。

## 4. 设计
- 关键设计决策：
  - 新增控件级 token：统一控制高度和圆角（`--gv-control-h`、`--gv-radius-sm`、`--gv-radius-md`）。
  - 提示词页标题区按钮改为两列等宽网格，保证视觉整齐。
  - 会话卡片操作区改为固定两列（下拉 + 固定宽按钮），避免右侧按钮宽度跳变。
  - 自定义下拉菜单改为相对触发器的固定锚点展开（绝对定位），减少漂移感。
  - 去掉多处 hover 的位移动画与阴影，避免“花哨且凌乱”。
- 取舍说明：
  - 视觉稳定和一致性优先，弱化卡片“浮起”动效。

## 5. 实现
- 主要变更文件：
  - `src/content/App.tsx`
  - `src/content/index.tsx`
- 核心逻辑：
  - `App.tsx`：`VoyagerSelect` 去掉动态 viewport 定位，简化为组件内固定锚点菜单。
  - `index.tsx`：统一按钮/输入/面板圆角和高度，重写提示词页标题区按钮布局、会话控制区网格、下拉菜单定位样式。
- 数据模型/存储变更：
  - 无。
- 配置/Manifest 变更：
  - 无。

## 6. 验证
- 自动化测试：
  - `npm run typecheck` 通过。
  - `npm run build` 通过。
- 手工验证步骤：
  1. 打开提示词页，检查标题区按钮是否等宽对齐。
  2. 检查筛选区、编辑区按钮是否与输入框高度一致。
  3. 在会话索引中观察“下拉 + 打开该文件夹”是否固定对齐。
  4. 展开任意下拉，确认弹层位置稳定且与触发器一致。
- 测试结果摘要：
  - 版面节奏更统一，按钮/下拉/输入框对齐显著改善。

## 7. 风险与回滚
- 已知风险：
  - 控件最小宽度策略在极窄屏会触发更多换行，已在移动断点下做降级处理。
- 回滚策略：
  - 回滚 `App.tsx` 的 `VoyagerSelect` 定位逻辑与 `index.tsx` 的全局控件样式改动。
