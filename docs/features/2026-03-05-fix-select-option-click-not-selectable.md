# 功能记录

## 1. 元信息
- 功能 ID：FIX-020
- 功能名称：修复下拉选项可见但无法点击选择
- 日期：2026-03-05
- 作者：Codex
- 分支：main

## 2. 概述
- 修复自定义下拉菜单中“选项显示但点击不生效”的问题。  
- 根因是组件级 `onBlurCapture` 在点击选项时抢先触发关闭，导致选项 `onClick` 无法完成选择。

## 3. 需求映射
- 对应 PRD 章节：`FR-01 侧边栏`
- 覆盖的用户场景：
  - 用户点击下拉选项可稳定完成选择并关闭菜单。
- 范围说明（In scope / Out of scope）：
  - In scope：下拉焦点关闭逻辑修复。
  - Out of scope：下拉样式与选项数据。

## 4. 设计
- 关键设计决策：
  - 移除 `VoyagerSelect` 的 `onBlurCapture` 自动关闭逻辑。
  - 保留外部点击关闭、`Esc` 关闭、`Tab` 关闭路径。

## 5. 实现
- 主要变更文件：
  - `src/content/App.tsx`
- 核心逻辑：
  - 删除 `gv-select-shell` 容器上的 `onBlurCapture` 回调，避免关闭时机早于选项点击。

## 6. 验证
- 自动化测试：
  - `npm run typecheck` 通过。
  - `npm run build` 通过。
- 手工验证步骤：

  2. 点击任意选项，确认值变更且菜单关闭。
  3. 再次打开并重复多次，确认稳定。
- 测试结果摘要：
  - 选项点击恢复正常。
