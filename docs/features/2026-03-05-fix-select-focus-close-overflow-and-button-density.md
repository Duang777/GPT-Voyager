# 功能记录

## 1. 元信息
- 功能 ID：FIX-019 / UI-018
- 功能名称：下拉可用性修复（Tab 收起 + 防裁切）与按钮密度对齐优化
- 日期：2026-03-05
- 作者：Codex
- 分支：main

## 2. 概述
- 针对本轮 bug 排查结果，完成三项修复：
  1. 下拉在键盘 `Tab` 导航离开时可自动收起。
  2. 会话滚动容器中的下拉菜单改为浮层定位，避免底部被裁切。
  3. 容易拥挤的按钮区取消统一最小宽度约束，缓解窄面板挤压和错位。

## 3. 需求映射
- 对应 PRD 章节：`FR-01 侧边栏`
- 覆盖的用户场景：
  - 键盘操作下拉时开关行为稳定且可预期。
  - 列表底部下拉展开不再被父容器截断。
  - 筛选/批量/内联操作按钮在窄面板中保持可读且不突兀换行。
- 范围说明（In scope / Out of scope）：
  - In scope：下拉交互、下拉菜单定位、按钮最小宽度策略。
  - Out of scope：业务逻辑与数据模型。

## 4. 设计
- 关键设计决策：
  - `VoyagerSelect` 增加焦点边界控制（`onBlurCapture`）与 `Tab` 收起逻辑。
  - 下拉菜单从容器内绝对定位改为视口浮层定位（`position: fixed` + 动态坐标），避免滚动裁切。
  - 针对高频操作区（`actions/filter/btn-row/batch-quick`）移除按钮最小宽度硬约束。
- 取舍说明：
  - 相比纯容器内定位，浮层定位在复杂滚动场景下更稳定；代价是需要维护滚动/缩放时的重定位监听。

## 5. 实现
- 主要变更文件：
  - `src/content/App.tsx`
  - `src/content/index.tsx`
- 核心逻辑：
  - `App.tsx`：
    - 为下拉增加菜单定位状态与同步函数。
    - 监听 `resize` 与捕获阶段 `scroll`，动态更新菜单坐标。
    - 增加 `onBlurCapture` 与 `Tab` 收起，完善键盘可用性。
  - `index.tsx`：
    - 下拉菜单样式改为 `position: fixed`。
    - 调整按钮最小宽度策略，提升窄宽度对齐表现。
- 数据模型/存储变更：
  - 无。
- 配置/Manifest 变更：
  - 无。

- 自动化测试：
  - `npm run typecheck` 通过。

## 7. 风险与回滚
- 已知风险：
  - 浮层定位依赖视口监听，若未来容器结构变化需复核定位计算。
- 回滚策略：
  - 回滚 `VoyagerSelect` 菜单定位与按钮最小宽度覆盖规则。
