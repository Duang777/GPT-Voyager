# FR-05：提示词片段库

## 1. 元信息
- 功能 ID：FEAT-004
- 功能名称：实现提示词片段库（保存/编辑/删除/复制/插入）
- 日期：2026-03-04
- 作者：Codex
- 分支：N/A（工作区直接更新）

## 2. 概述
在侧边栏新增提示词库模块，支持维护常用提示词模板。用户可新增、编辑、删除提示词，支持一键复制到剪贴板和一键插入当前 ChatGPT 输入框。

## 3. 需求映射
- 对应 PRD 章节：6.1（P0）、8（FR-05）、10（技术方案）、15（DoD）
- 覆盖的用户场景：
  - 用户希望将高频提示词沉淀为可复用模板。
  - 用户希望在聊天页面内快速复制或直接插入提示词。
- 本次范围：
  - 提示词本地存储与加载
  - 提示词 CRUD（新增、编辑、删除）
  - 一键复制
  - 一键插入输入框（textarea/contenteditable）
- 本次不包含：
  - 提示词分组和标签
  - 提示词云端同步
  - 版本历史

## 4. 设计
- 关键设计决策：
  - 使用本地存储键 `gpt_voyager_prompt_library_v1` 保存提示词。
  - 数据结构：`PromptSnippet(id/title/content/createdAt/updatedAt)`。
  - 输入框插入优先匹配 ChatGPT `textarea`，再回退到 `contenteditable`。
  - 状态提示使用短暂 toast 文本（已复制/已插入/失败）。
- 取舍说明：
  - 当前采用“追加插入”策略，避免覆盖用户已有输入。
  - 当前不做模板变量系统，先保证稳定可用。
- 权限/数据/隐私考虑：
  - 数据仅存储在本地 `chrome.storage.local`。
  - 复制依赖浏览器剪贴板 API，不新增扩展权限。

## 6. 验证
- 自动化测试：
  - `npm run typecheck`
  - `npm run build`
- 手工验证步骤：
  1. 在扩展页刷新 `dist/` 后打开 ChatGPT 页面
  2. 新建一条提示词（标题 + 正文）并保存
  3. 点击“编辑”修改内容并更新
  4. 点击“复制”并在外部输入框粘贴验证
  5. 点击“插入”，确认 ChatGPT 输入框新增该提示词内容
  6. 删除提示词并刷新页面，确认列表状态正确持久化
- 测试结果摘要：
  - 类型检查与构建通过，核心手工路径可用。

## 7. 风险与回滚
- 已知风险：
  - ChatGPT 输入区域结构变化时，插入逻辑可能失效。
  - 剪贴板权限受浏览器策略影响，少数场景可能复制失败。
- 回滚策略：
  - 回退 `promptLibrary.ts` 与 `App.tsx` 到 FEAT-003 版本。
