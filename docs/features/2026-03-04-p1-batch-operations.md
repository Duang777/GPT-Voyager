# FEAT-020：P1 会话批量操作（多选 + 批量标签/文件夹）

## 1. 元信息
- 功能 ID：FEAT-020
- 功能名称：实现会话多选与批量管理能力
- 日期：2026-03-04
- 作者：Codex
- 分支：N/A（工作区直接更新）

## 2. 概述
继续按顺序开发，补齐 P1 的批量管理核心能力。在会话索引中新增多选机制和批量操作面板，支持对选中会话批量设置文件夹、批量添加标签和批量移除标签。

## 3. 需求映射
- 对应 PRD 章节：6.2（P1：批量操作）、8（FR-03 分类管理）
- 覆盖的用户场景：
  - 用户需要一次性整理大量会话，减少重复点击。
- 本次范围：
  - 会话条目多选
  - 全选当前筛选结果 / 清空选择
  - 批量设文件夹
  - 批量加标签 / 去标签
- 本次不包含：
  - 批量删除会话
  - 撤销栈

## 4. 设计
- 关键设计决策：
  - 使用独立 `selectedConversationIds` 状态维护批量选择。
  - 批量更新复用 `upsertConversationMeta`，保证与现有元数据一致性。
  - 批量面板放在会话索引筛选区下方，保持操作语境一致。
- 取舍说明：
  - 暂不做复杂“跨筛选条件累积选择”，先保证当前筛选集可批量处理。
- 权限/数据/隐私考虑：
  - 仅本地状态与 `chrome.storage.local` 写入，不涉及外发。

## 6. 验证
- 自动化测试：
  - `npm run typecheck`
  - `npm run build`
- 手工验证步骤：
  1. 勾选多条会话，选择文件夹并执行“批量设文件夹”。
  2. 选择标签后执行“批量加标签 / 批量去标签”。
  3. 切换筛选条件后再次批量操作，确认状态与结果正确。
- 测试结果摘要：
  - 类型检查与构建通过，批量流程可用。

## 7. 风险与回滚
- 已知风险：
  - 用户可能误操作批量修改，当前仅有状态提示无撤销。
- 回滚策略：
  - 回退 `App.tsx` 批量状态和批量面板 UI。
