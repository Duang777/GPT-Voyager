# 功能记录

## 1. 元信息
- 功能 ID：FEAT-050
- 功能名称：会话工作台卡片化入口与单模块视图
- 日期：2026-03-05
- 作者：Codex
- 分支：main

## 2. 概述
- 针对“会话面板内容堆叠、阅读复杂”的问题，将会话工作台改为“入口卡片 -> 进入单一子工作台”的组织方式。用户先看到卡片总览，点击后只显示对应模块，避免多个工作台同时挤在一页造成认知负担。

## 3. 需求映射
- 对应 PRD 章节：`FR-01 侧边栏`、`FR-02 会话索引`
- 覆盖的用户场景：
  - 用户进入会话工作台时，先看功能入口，再进入某个单独模块深度操作。
  - 用户希望快速切换模块并返回入口，不被“全量信息”淹没。
- 范围说明（In scope / Out of scope）：
  - In scope：会话工作台 UI 结构与可见性控制。
  - Out of scope：模块内部功能逻辑重写。

## 4. 设计
- 关键设计决策：
  - 新增“工作台入口”卡片网格作为一级导航。
  - 模块区域使用 `workspace mode` 显示规则：同一时间只显示一个子工作台。
  - 保留原有模块实现，仅调整组织与展示层。
- 取舍说明：
  - 首次进入需要一次点击进入模块，但整体信息密度显著下降，易读性更高。
- 权限/数据/隐私考虑：
  - 纯前端 UI 调整，不涉及权限和存储模型变化。

## 5. 实现
- 主要变更文件：
  - `src/content/App.tsx`
  - `src/content/index.tsx`
  - `PRD.md`
- 核心逻辑：
  - 新增 `ConversationWorkspaceKey` 与 `conversationWorkspace` 状态。
  - 新增工作台入口卡片数据与“返回入口”动作。
  - 为总览/导出/星标/时间线/公式/Mermaid/分类/索引区块添加 `data-gv-workspace` 标识。
  - 通过样式规则实现“按当前 mode 仅显示目标区块”。
- 数据模型/存储变更：
  - 无。
- 配置/Manifest 变更：
  - 无。

## 6. 验证
- 自动化测试：
  - `npm run typecheck` 通过。
  - `npm run build` 通过。
- 手工验证步骤：
  1. 打开会话工作台，确认默认显示“工作台入口”卡片区。
  2. 点击任一卡片（如“会话索引”），确认仅显示该模块内容。
  3. 点击“返回入口”，确认回到卡片总览。
  4. 切换到其他主导航（提示词库/设置）再返回，入口状态符合预期。
- 测试结果摘要：
  - 编译通过；工作台已从“堆叠式”改为“卡片入口 + 单模块视图”。

## 7. 风险与回滚
- 已知风险：
  - 用户需要适应新的入口路径（先点卡片再进模块）。
- 回滚策略：
  - 回滚 `App.tsx` 与 `index.tsx` 的工作台模式和样式控制改动，恢复原先全部模块同页展示。

## 8. 后续事项
- 待办项：
  - 在入口卡片上增加“最近访问模块”置顶能力。
  - 支持记忆上次打开的子工作台（可在设置中开关）。
- 潜在优化：
  - 卡片支持拖拽排序，按用户习惯定制入口布局。
