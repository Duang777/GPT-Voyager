# 功能记录

## 1. 元信息
- 功能 ID：UI-014 / FIX-014
- 功能名称：全局下拉框（Select）极简重构（全页面统一）
- 日期：2026-03-05
- 作者：Codex
- 分支：main

## 2. 概述
- 针对“下拉框很丑、每个页面都一样突兀”的反馈，对扩展内所有 `select` 控件做统一轻量化重构。  
- 重点去除厚重焦点圈和过于显眼的胶囊框感，改为更克制的细边框、低干扰焦点与紧凑尺寸。

## 3. 需求映射
- 对应 PRD 章节：`FR-01 侧边栏`、`FR-02 会话索引`、`FR-07 设置`
- 覆盖的用户场景：
  - 用户在会话索引、批量操作、会话卡片、设置中心等区域使用下拉时，视觉保持统一且不抢眼。
  - 鼠标点击展开时不再出现“粗框 + 强高亮”的突兀感。
- 范围说明（In scope / Out of scope）：
  - In scope：`select` 全局样式、hover/focus 可视反馈、局部紧凑尺寸适配。
  - Out of scope：下拉数据项逻辑、筛选业务行为。

## 4. 设计
- 关键设计决策：
  - 统一 `gv-select` 边框为更浅灰细线，圆角和高度略收敛。
  - `focus` 与 `focus-visible` 分离：点击时弱化，键盘可访问时保留适度焦点环。
  - 会话卡片控制区与批量块内的下拉采用更紧凑尺寸，降低组件堆叠感。
  - 统一 `option` 文本与背景基础样式，避免深浅不一致。
- 取舍说明：
  - 保留原生 `select` 行为稳定性，不回到自定义浮层方案。
- 权限/数据/隐私考虑：
  - 纯样式改造，不涉及权限与数据。

## 5. 实现
- 主要变更文件：
  - `src/content/index.tsx`
- 核心逻辑：
  - 调整 `.gv-select` 的边框/圆角/高度/箭头尺寸/hover/focus 样式。
  - 新增 `.gv-item-controls .gv-select`、`.gv-batch-block-row .gv-select` 的紧凑尺寸规则。
  - 调整全局焦点规则：`select` 使用更轻焦点环，避免与输入框/按钮相同强度。
- 数据模型/存储变更：
  - 无。
- 配置/Manifest 变更：
  - 无。

## 6. 验证
- 自动化测试：
  - `npm run typecheck` 通过。
  - `npm run build` 通过。
- 手工验证步骤：
  1. 打开会话索引，观察排序/密度下拉外观。
  2. 展开批量操作，观察文件夹/标签下拉与按钮对齐。
  3. 在会话卡片内查看文件夹下拉，确认边框与焦点不突兀。
  4. 切换到设置中心，确认下拉样式与其它页面一致。
- 测试结果摘要：
  - 下拉框视觉更轻、更统一，强高亮感显著降低。

## 7. 风险与回滚
- 已知风险：
  - 原生下拉弹出层（系统层）在不同操作系统仍有默认差异，属于浏览器原生行为。
- 回滚策略：
  - 回滚 `index.tsx` 中 `.gv-select` 与 `focus-visible` 相关规则。

## 8. 后续事项
- 待办项：
  - 对导航按钮和操作按钮做同级别“轻描边”统一，进一步减少视觉噪声。
- 潜在优化：
  - 增加“超紧凑控件模式”设置项，按用户偏好降低组件高度。
