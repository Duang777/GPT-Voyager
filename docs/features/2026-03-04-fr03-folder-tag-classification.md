# FR-03：文件夹/标签分类管理

## 1. 元信息
- 功能 ID：FEAT-003
- 功能名称：实现文件夹/标签管理、会话归类与按分类筛选（FR-03）
- 日期：2026-03-04
- 作者：Codex
- 分支：N/A（工作区直接更新）

## 2. 概述
在现有会话索引基础上，新增文件夹与标签分类能力。用户可以创建/删除文件夹和标签，将会话分配到文件夹并绑定多个标签，同时支持按文件夹和标签快速筛选会话列表。

## 3. 需求映射
- 对应 PRD 章节：6.1（P0）、8（FR-03）、10（技术方案）、15（DoD）
- 覆盖的用户场景：
  - 用户希望将会话组织到文件夹和标签中做长期管理。
  - 用户希望通过文件夹/标签快速定位目标会话。
- 本次范围：
  - 新增分类数据存储层（本地持久化）
  - 新增文件夹/标签创建与删除
  - 新增会话级文件夹分配与标签切换
  - 新增“按文件夹/按标签”筛选
- 本次不包含：
  - 批量分类（P1）
  - 标签颜色和排序规则自定义
  - 云同步

## 4. 设计
- 关键设计决策：
  - 分类状态独立存储，键名为 `gpt_voyager_classification_v1`。
  - 会话元数据结构：`{ folderId?: string, tagIds: string[] }`。
  - 清理策略：删除文件夹/标签时自动清理对应会话绑定。
  - 输入策略：名称归一化并进行重复校验。
- 取舍说明：
  - 当前仅支持创建和删除，不包含重命名，减少状态迁移复杂度。
  - 当前按静态顺序展示分类项，后续再加入拖拽排序。
- 权限/数据/隐私考虑：
  - 分类数据仅存储于 `chrome.storage.local`，不外发。
  - 未新增扩展权限。
