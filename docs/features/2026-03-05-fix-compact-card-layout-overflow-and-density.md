# 功能记录

## 1. 元信息
- 功能 ID：FIX-023 / UI-020
- 功能名称：修复紧凑会话卡片排版溢出与密度失衡
- 日期：2026-03-05
- 作者：Codex
- 分支：main

## 2. 概述
- 修复“紧凑卡片排版有问题”的反馈，主要表现为时间文本易被截断、控件横向拥挤、紧凑模式与虚拟行高估算不一致。  
- 本次将紧凑模式改为更稳的单列操作区，并同步调整虚拟滚动紧凑行高估算。

## 3. 需求映射
- 对应 PRD 章节：`FR-02 会话索引`
- 覆盖的用户场景：
  - 紧凑模式下会话卡片信息不再拥挤错位。
  - 时间信息在窄宽度下可保持可读，不易被截断。
- 范围说明（In scope / Out of scope）：
  - In scope：紧凑模式卡片样式、虚拟滚动紧凑行高估算。
  - Out of scope：会话筛选和元数据逻辑。

## 4. 设计
- 关键设计决策：
  - 紧凑模式下 `item-controls` 改为单列，避免下拉与按钮并排挤压。
  - 元信息行改为可换行 `flex`，时间戳保留右对齐优先。
  - 下拉、星标、备注输入在紧凑模式统一更小高度。
  - 紧凑模式虚拟行高估算由 `150` 提高到 `174`，减少视觉高度与估算不一致造成的滚动错位感。
- 取舍说明：
  - 操作区单列会牺牲部分横向密度，但明显提升稳定性与可读性。

## 5. 实现
- 主要变更文件：
  - `src/content/index.tsx`
  - `src/content/App.tsx`
- 核心逻辑：
  - `index.tsx`：重写 `.gv-list-compact` 下的 `item/meta/controls/tag/note` 样式。
  - `App.tsx`：调整 `CONVERSATION_ROW_HEIGHT_COMPACT` 常量。
- 数据模型/存储变更：
  - 无。
- 配置/Manifest 变更：
  - 无。

## 6. 验证
- 自动化测试：
  - `npm run typecheck` 通过。
  - `npm run build` 通过。
- 手工验证步骤：
  1. 在会话索引切换“卡片密度=紧凑”。
  2. 检查每张卡片的时间、操作区、标签区、备注输入是否出现拥挤与截断。
  3. 滚动列表观察虚拟滚动是否平稳。
- 测试结果摘要：
  - 紧凑卡片布局更规整，溢出与挤压现象显著缓解。

## 7. 风险与回滚
- 已知风险：
  - 紧凑模式高度略增，单屏可见项可能略少于旧实现。
- 回滚策略：
  - 回滚 `index.tsx` 的 `.gv-list-compact` 规则与 `App.tsx` 紧凑行高常量。
