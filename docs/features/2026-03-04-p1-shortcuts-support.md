# FEAT-011：P1 快捷键支持（可开关）

## 1. 元信息
- 功能 ID：FEAT-011
- 功能名称：实现快捷键操作与设置开关
- 日期：2026-03-04
- 作者：Codex
- 分支：N/A（工作区直接更新）

## 2. 概述
在不改变现有主流程的前提下，新增常用快捷键以提升高频操作效率，并允许用户在设置中关闭快捷键，避免与个人系统/浏览器习惯冲突。

## 3. 需求映射
- 对应 PRD 章节：6.2（P1）、8（FR-07 设置扩展）、9（交互效率）
- 覆盖的用户场景：
  - 用户希望无需鼠标即可快速开关面板、扫描、导出。
- 本次范围：
  - 新增快捷键监听与动作映射
  - 新增快捷键启用开关设置
- 本次不包含：
  - 快捷键自定义配置
  - 批量操作的快捷键映射

## 4. 设计
- 关键设计决策：
  - 快捷键方案：`Ctrl/⌘ + Shift + K`（开关面板）、`Ctrl/⌘ + Shift + R`（重新扫描）、`Ctrl/⌘ + Shift + E`（快速导出）。
  - 在输入框/可编辑区域内禁用快捷键触发，避免打字冲突。
  - 通过设置项 `enableShortcuts` 进行总开关控制。
- 取舍说明：
  - 先提供固定快捷键，保证实现简单稳定；后续再做可配置化。
- 权限/数据/隐私考虑：
  - 仅页面内监听键盘事件，不新增扩展权限。

## 5. 实现
- 主要变更文件：
  - `src/content/App.tsx`
  - `src/content/settingsStore.ts`
  - `README.md`
- 核心逻辑：
  - `App.tsx` 新增键盘监听 effect：
    - 过滤编辑态目标元素
    - 识别组合键后执行对应操作
  - `settingsStore.ts` 新增字段：`enableShortcuts`（默认 `true`）
  - 设置中心新增快捷键开关 UI。
- 数据模型/存储变更：
  - `gpt_voyager_settings_v1` 结构新增 `enableShortcuts: boolean`
- 配置/Manifest 变更：
  - 无

## 6. 验证
- 自动化测试：
  - `npm run typecheck`
  - `npm run build`
- 手工验证步骤：
  1. 开启快捷键，在非输入区按 `Ctrl/⌘ + Shift + K`，确认面板开关。
  2. 按 `Ctrl/⌘ + Shift + R`，确认触发重新扫描并显示状态。
  3. 按 `Ctrl/⌘ + Shift + E`，确认执行默认导出格式。
  4. 关闭“启用快捷键”，重复以上操作，应不再触发。
- 测试结果摘要：
  - 类型检查和构建通过，快捷键逻辑可用。

## 7. 风险与回滚
- 已知风险：
  - 个别系统快捷键可能与浏览器插件冲突。
- 回滚策略：
  - 回退 `App.tsx` 快捷键 effect 与 `settingsStore.ts` 新字段。
