# FEAT-014：P1 提示词标签分组与搜索筛选

## 1. 元信息
- 功能 ID：FEAT-014
- 功能名称：实现提示词标签分组、标签筛选与全文搜索
- 日期：2026-03-04
- 作者：Codex
- 分支：N/A（工作区直接更新）

## 2. 概述
继续按顺序推进 P1 能力，在提示词库中加入“标签分组 + 搜索”机制。现在每条提示词可以维护标签，列表支持按标签筛选、按标题/正文/标签检索，提升模板数量增加后的查找效率。

## 3. 需求映射
- 对应 PRD 章节：6.2（P1：提示词分组与变量模板，先完成分组部分）、8（FR-05 提示词库）
- 覆盖的用户场景：
  - 用户积累大量提示词后，希望快速按主题标签聚合查看。
  - 用户希望通过关键词快速定位历史模板。
- 本次范围：
  - 提示词数据模型新增 `tags`
  - 新增标签输入、标签筛选、关键词检索
  - 标签在列表项内可点击反向筛选
- 本次不包含：
  - 提示词变量模板（如 `{{topic}}` 占位符）
  - 标签层级与颜色管理

## 4. 设计
- 关键设计决策：
  - 标签存储为 `string[]`，统一走 `normalizePromptTags` 做清洗与去重。
  - 兼容历史数据：老提示词没有 `tags` 字段时自动回填为空数组。
  - 标签最多保留 12 个，单标签长度限制 24 字，避免 UI 过载。
- 取舍说明：
  - 使用逗号分隔输入，学习成本低；暂不引入复杂标签编辑器。
- 权限/数据/隐私考虑：
  - 仅本地 `chrome.storage.local` 读写，不新增外部请求。

## 6. 验证
- 自动化测试：
  - `npm run typecheck`
  - `npm run build`
- 手工验证步骤：
  1. 新建提示词并填写标签，保存后确认标签展示在卡片中。
  2. 输入关键词搜索（标题/正文/标签），确认结果正确。
  3. 通过下拉“全部标签”选择某标签，确认只显示对应模板。
  4. 点击卡片内标签，确认自动切换到该标签筛选。
  5. 编辑已有提示词后刷新页面，确认标签与内容均持久化。
- 测试结果摘要：
  - 类型检查与构建通过，分组与搜索链路可用。
