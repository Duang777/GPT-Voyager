# 功能记录

## 1. 元信息
- 功能 ID：UI-015 / FIX-015
- 功能名称：全页面统一自定义下拉（替换原生 Select 弹层）
- 日期：2026-03-05
- 作者：Codex
- 分支：main

## 2. 概述
- 针对“下拉展开后仍是系统蓝色高亮、风格不一致”的问题，将扩展内所有原生 `select` 全量替换为统一的自定义下拉组件。  
- 通过自定义触发器 + 自定义菜单弹层，彻底避免浏览器原生菜单样式差异（蓝色选中条、字体/边框不一致）。

## 3. 需求映射
- 对应 PRD 章节：`FR-01 侧边栏`、`FR-02 会话索引`、`FR-05 提示词库`、`FR-07 设置`
- 覆盖的用户场景：
  - 时间线、公式、会话索引、批量操作、会话卡片、提示词筛选、设置中心等页面下拉样式一致。
  - 展开菜单时不再出现系统默认蓝色选中条。
- 范围说明（In scope / Out of scope）：
  - In scope：下拉组件实现、弹层定位、键盘交互、全页面替换。
  - Out of scope：筛选逻辑、设置项业务逻辑。

## 4. 设计
- 关键设计决策：
  - 新增 `VoyagerSelect` 统一组件（按钮触发 + portal 弹层菜单）。
  - 菜单使用 `position: fixed` + 动态定位，避免受滚动容器裁切。
  - 支持外部点击关闭、`Esc` 关闭、上下箭头切换选项。
  - 提供紧凑版本样式 `gv-select-shell-compact` 适配会话卡片/批量区。
- 取舍说明：
  - 以一致视觉为优先，放弃原生下拉在系统级菜单交互上的默认外观。
- 权限/数据/隐私考虑：
  - 仅 UI 组件替换，不涉及权限与数据存储。

## 5. 实现
- 主要变更文件：
  - `src/content/App.tsx`
  - `src/content/index.tsx`
- 核心逻辑：
  - 在 `App.tsx` 中新增 `VoyagerSelect` 组件与通用选项常量。
  - 将原有所有 `<select>` 替换为 `VoyagerSelect`（时间线、公式、索引、批量、会话卡片、提示词、设置）。
  - 在 `index.tsx` 中新增统一样式类：`gv-select-shell`、`gv-select-trigger`、`gv-select-menu`、`gv-select-option`。
  - 调整焦点规则，避免通用按钮焦点样式覆盖自定义下拉触发器。
- 数据模型/存储变更：
  - 无。
- 配置/Manifest 变更：
  - 无。

## 6. 验证
- 自动化测试：
  - `npm run typecheck` 通过。
  - `npm run build` 通过。
- 手工验证步骤：
  1. 在会话索引页面展开排序/密度、分类筛选、批量操作中的下拉。
  2. 在会话卡片展开文件夹下拉并切换。
  3. 在时间线、公式、提示词、设置中心分别展开下拉，确认风格一致。
  4. 检查展开列表不再出现系统蓝色高亮菜单风格。
- 测试结果摘要：
  - 全页面下拉交互与视觉统一，原生系统菜单风格差异已消除。

## 7. 风险与回滚
- 已知风险：
  - 自定义弹层需要持续关注极端滚动场景下的定位表现。
- 回滚策略：
  - 回滚 `App.tsx` 的 `VoyagerSelect` 与相关替换代码，恢复原生 `select` 渲染路径。
