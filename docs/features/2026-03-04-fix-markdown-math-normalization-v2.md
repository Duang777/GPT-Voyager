# 修复：Markdown 公式归一化（二次）

## 1. 元信息
- 功能 ID：FIX-004
- 功能名称：修复 `\\[...\\]` 定界符与双反斜杠命令导致的 Markdown 公式解析错误
- 日期：2026-03-04
- 作者：Codex
- 分支：N/A（工作区直接更新）

## 2. 概述
针对“HTML 完美但 Markdown 仍有局部公式渲染错误”的反馈，继续增强数学归一化逻辑。重点解决：  
1) 公式块使用 `\\[` / `\\]` 双反斜杠定界符无法被识别；  
2) 公式体里出现 `\\begin` 等双反斜杠命令导致 KaTeX parse error。

## 3. 需求映射
- 对应 PRD 章节：8（FR-06）、15（DoD）
- 覆盖的用户场景：
  - 用户导出的 Markdown 在数学渲染器中报错更少。
- 本次范围：
  - 定界符匹配支持 `\[` 和 `\\[` 两种写法
  - 数学体内命令恢复（`\\command` -> `\command`）
  - 数学体内下划线转义回退（`\_` -> `_`）
- 本次不包含：
  - 所有非标准 LaTeX 的自动纠错

## 4. 设计
- 关键设计决策：
  - 仅在数学定界符内部做命令归一化，不影响普通文本。
  - 保持代码块隔离，避免代码段误转换。
- 取舍说明：
  - 以“减少误判”为优先，不做激进全局替换。
- 权限/数据/隐私考虑：
  - 无新增权限、无新增存储。

## 5. 实现
- 主要变更文件：
  - `src/content/conversationExport.ts`
- 核心逻辑：
  - `normalizeLatexDelimiters` 匹配升级为 `\\{1,2}` 级别
  - 新增 `normalizeMathBody` 对数学体进行命令和下划线清洗
  - 移除上一版中高风险的全局下划线处理链路
- 数据模型/存储变更：
  - 无
- 配置/Manifest 变更：
  - 无

## 6. 验证

  - `npm run typecheck`

- 手工验证步骤：
  1. 导出包含 `\\[ ... \\]`、`\\begin{cases}` 的会话 Markdown
  2. 在支持数学渲染的 Markdown 查看器中打开
  3. 验证 parse error 是否下降
- 测试结果摘要：
