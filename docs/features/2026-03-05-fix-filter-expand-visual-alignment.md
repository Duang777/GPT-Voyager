# 功能记录

## 1. 元信息
- 功能 ID：FIX-010
- 功能名称：筛选展开控件视觉对齐修复（下拉不再挤压变形）
- 日期：2026-03-05
- 作者：Codex
- 分支：main

## 2. 概述
- 修复会话时间线筛选区在窄面板中的“展开很丑”问题：下拉控件被按钮挤压后出现文字竖排、按钮高度不一致、展开层观感突兀。调整为更稳定的两列布局和紧凑下拉样式，保证展开前后视觉一致。

## 3. 需求映射
- 对应 PRD 章节：`FR-01 侧边栏`
- 覆盖的用户场景：
  - 用户在会话工作台打开筛选下拉时，界面保持简洁对齐，不出现变形。
- 范围说明（In scope / Out of scope）：
  - In scope：筛选行布局、InlineSelect 按钮与下拉层样式。
  - Out of scope：筛选逻辑和数据行为。

## 4. 设计
- 关键设计决策：
  - `gv-filter-row` 改为两列网格，避免 `auto auto` 按钮压缩下拉宽度。
  - 筛选行按钮宽度统一为 `100%`，形成稳定按钮矩阵。
  - InlineSelect 按钮固定高度、文本单行省略、箭头固定尺寸。
  - 展开态边框高亮减弱，避免粗描边突兀感。
- 取舍说明：
  - 操作按钮从同一行改为下一行并列，更稳更整齐，但垂直占用略增。
- 权限/数据/隐私考虑：
  - 仅样式调整，不涉及数据与权限。

## 5. 实现
- 主要变更文件：
  - `src/content/index.tsx`
- 核心逻辑：
  - 修改筛选行网格模板和按钮宽度规则。
  - 优化 `.gv-inline-select-btn`、`.gv-inline-select-menu` 的尺寸和可读性样式。
- 数据模型/存储变更：
  - 无。
- 配置/Manifest 变更：
  - 无。

## 6. 验证
- 自动化测试：
  - `npm run typecheck` 通过。
  - `npm run build` 通过。
- 手工验证步骤：
  1. 打开会话时间线模块。
  2. 点击角色筛选和标签筛选下拉。
  3. 检查文字是否保持单行、按钮高度是否一致、展开层是否对齐。
- 测试结果摘要：
  - 下拉展开视觉稳定，控件不再出现竖排压缩与错位。

- 回滚策略：
