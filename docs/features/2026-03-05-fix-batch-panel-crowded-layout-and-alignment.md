# 功能记录

## 1. 元信息
- 功能 ID：FIX-022 / UI-019
- 功能名称：修复批量操作区控件挤压与对齐混乱
- 日期：2026-03-05
- 作者：Codex
- 分支：main

## 2. 概述
- 针对批量操作区“文件夹/标签块挤在一起、按钮和下拉拥挤”的问题，重排为更稳定的纵向结构。  
- 目标是降低横向挤压，确保在当前侧栏宽度下仍保持清晰可点。

## 3. 需求映射
- 对应 PRD 章节：`FR-02 会话索引`
- 覆盖的用户场景：
  - 用户在批量操作区设置文件夹/标签时，控件不再出现局部挤压与错位。
- 范围说明（In scope / Out of scope）：
  - In scope：批量区布局样式与按钮宽度规则。
  - Out of scope：批量操作逻辑本身。

## 4. 设计
- 关键设计决策：
  - `gv-batch-grid` 改为单列，避免两块并排造成窄宽度压缩。
  - `gv-batch-block-row` 改为单列流，先下拉后按钮，提升可读性和点击容错。
  - 标签操作按钮改为等宽双列网格；“应用”按钮改为全宽按钮。
- 取舍说明：
  - 相比并排布局，单列会增加垂直占用，但在窄侧栏下显著提升可用性。

## 5. 实现
- 主要变更文件：
  - `src/content/index.tsx`
- 核心逻辑：
  - 调整 `gv-batch-grid` 列规则与 `gv-batch-block-row` 网格规则。
  - 新增批量区按钮全宽和双列网格对齐样式。

## 6. 验证
- 自动化测试：
  - `npm run typecheck` 通过。
  - `npm run build` 通过。
- 手工验证步骤：
  1. 打开会话索引，展开批量操作。
  2. 检查文件夹块与标签块是否改为纵向、是否无挤压。
  3. 验证“应用/添加/移除”按钮是否易点且对齐。
- 测试结果摘要：
  - 批量区控件拥挤问题明显缓解，版面更规整。

- 回滚策略：
