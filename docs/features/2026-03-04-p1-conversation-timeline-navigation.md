# FEAT-017：P1 会话时间线与节点定位

## 1. 元信息
- 功能 ID：FEAT-017
- 功能名称：实现会话时间线导航（筛选 + 点击定位）
- 日期：2026-03-04
- 作者：Codex
- 分支：N/A（工作区直接更新）

## 2. 概述
继续按顺序推进 P1，新增“会话时间线”模块。插件会读取当前会话中的消息节点，形成时间线列表，支持关键词和角色筛选，并可点击任意节点直接滚动定位到对应消息。

## 3. 需求映射
- 对应 PRD 章节：6.2（P1）、8（FR-04 检索体验增强）
- 覆盖的用户场景：
  - 长对话下快速跳转到某一轮关键消息。
  - 仅查看用户消息/助手消息，减少阅读成本。
- 本次范围：
  - 时间线采集与观察
  - 角色筛选、关键词筛选
  - 节点点击定位与高亮反馈
- 本次不包含：
  - 节点级导出
  - 时间线折叠分组

## 4. 设计
- 关键设计决策：
  - 新建 `conversationTimeline.ts`，将 DOM 采集与观察从 `App.tsx` 拆出。
  - 采集基于 `[data-message-author-role]`，兼容 ChatGPT 常见消息容器结构。
  - 点击定位后用临时 outline 高亮目标节点，避免侵入页面样式系统。
- 取舍说明：
  - 采用轻量预览文本（截断）而非完整正文，保证列表性能。
- 权限/数据/隐私考虑：
  - 仅读取当前页面 DOM，不新增权限，不上传数据。

## 5. 实现
- 主要变更文件：
  - `src/content/conversationTimeline.ts`
  - `src/content/App.tsx`
  - `src/content/index.tsx`

  - `App.tsx` 新增时间线 UI：
    - 搜索输入
    - 角色下拉筛选
    - 列表点击定位
- 数据模型/存储变更：
  - 无新增持久化字段（时间线为运行时派生数据）。
- 配置/Manifest 变更：
  - 无

## 6. 验证
- 自动化测试：
  - `npm run typecheck`
  - `npm run build`
- 手工验证步骤：
  1. 打开任意 `chatgpt.com/c/...` 对话。
  2. 在“会话时间线”中搜索关键词并切换角色筛选。
  3. 点击任一时间线条目，确认页面平滑滚动并高亮对应消息。
  4. 继续对话生成新消息，确认时间线自动更新。
- 测试结果摘要：
