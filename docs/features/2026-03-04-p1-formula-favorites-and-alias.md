# FEAT-025：P1 公式收藏与别名管理

## 1. 元信息
- 功能 ID：FEAT-025
- 功能名称：公式收藏与别名管理
- 日期：2026-03-04
- 作者：Codex
- 分支：N/A

## 2. 概述
在“公式工作台复制与定位”基础上，新增公式收藏能力。用户可以将当前会话中的公式加入收藏，给收藏公式设置可读别名，并在收藏列表中进行搜索、复制、定位来源会话或删除，解决公式复用场景下“重复查找、记不住来源”的痛点。

## 3. 需求映射
- 对应 PRD 章节：
  - 6.2（P1：公式收藏与别名管理）
  - 8（FR-06 导入导出）
- 覆盖的用户场景：
  - 用户在长会话中看到常用公式，希望一键收藏并后续快速复用。
  - 用户希望通过自定义别名快速识别公式含义，而不是只看 TeX。
  - 用户希望收藏数据可随 JSON 备份一起导出导入。
- 范围说明（In scope / Out of scope）：
  - In scope：
    - 公式收藏/取消收藏
    - 收藏别名编辑
    - 收藏搜索、复制、定位来源、删除
    - 收藏数据本地持久化与备份导入导出
  - Out of scope：
    - 收藏分组/标签体系
    - 收藏跨设备同步
    - 收藏导出为独立文件

## 4. 设计
- 关键设计决策：
  - 以 `sourceConversationId + displayMode + tex` 作为去重匹配键，避免同会话重复收藏相同公式。
  - 别名默认由 TeX 自动截断生成，支持后续用户覆盖编辑。
  - 收藏列表放在“公式工作台”内部，保持操作语境一致。
  - 收藏纳入现有 JSON 备份模型，保证数据闭环。
- 取舍说明：
  - 先不做复杂收藏组织结构（文件夹/标签），优先交付可用的别名与复用链路。
  - 来源定位优先处理“当前会话定位”，跨会话场景直接打开来源会话页面。
- 权限/数据/隐私考虑：
  - 所有收藏数据仅存储于 `chrome.storage.local`。
  - 无额外网络请求，无新增扩展权限。

## 6. 验证
- 自动化测试：
  - `npm run typecheck`：通过
  - `npm run build`：通过
- 手工验证步骤：
  1. 在公式工作台中点击“收藏公式”，观察公式状态变化为“已收藏”。
  2. 在“公式收藏”列表中修改别名并搜索别名，确认过滤正确。
  3. 点击“复制 TeX”，确认复制反馈正常。
  4. 点击“定位来源/打开来源会话”，确认行为符合当前会话与跨会话逻辑。
  5. 导出 JSON 备份并再次导入，确认收藏数据可恢复。
- 测试结果摘要：
  - 类型检查与构建通过，收藏与别名管理链路可用。

## 7. 风险与回滚
- 已知风险：
  - 公式去重基于文本归一化，极少数格式差异公式可能被视为不同条目。
  - 跨会话定位依赖会话 URL 与 ID 可访问性，部分历史会话可能打不开。
- 回滚策略：
  - 回退 `formulaFavoritesStore.ts` 与 `App.tsx` 收藏相关状态和 UI。
  - 回退 `dataBackup.ts` 的 `formulaFavorites` 字段扩展。
  - 回退样式与文档同步内容。
