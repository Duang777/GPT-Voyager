# FEAT-023：P1 会话时间线按节点导出（Markdown/HTML）

## 1. 元信息
- 功能 ID：FEAT-023
- 功能名称：会话时间线按节点导出（Markdown/HTML）
- 日期：2026-03-04
- 作者：Codex
- 分支：N/A

## 2. 概述
在已有“会话时间线导航”的基础上，新增按当前筛选结果导出时间线节点的能力。用户可以将时间线中可见节点直接导出为 Markdown 或 HTML，便于复盘长对话、归档关键节点和对外分享结构化记录。

## 3. 需求映射
- 对应 PRD 章节：
  - 6.2（P1：会话时间线按节点导出）
  - 8（FR-06 导出/导入）
- 覆盖的用户场景：
  - 用户已通过“角色 + 关键词”筛出关键消息节点，希望只导出这一部分内容。
  - 用户希望延续现有会话导出格式，保留公式、代码块等内容结构。
- 范围说明（In scope / Out of scope）：
  - In scope：
    - 时间线筛选结果导出为 Markdown
    - 时间线筛选结果导出为 HTML
    - 时间线区域新增“快速导出时间线/时间线 MD/时间线 HTML”按钮
  - Out of scope：
    - 导出为 PDF/Docx
    - 跨会话批量时间线导出
    - 时间线导出模板自定义

## 4. 设计
- 关键设计决策：
  - 导出对象采用“当前时间线筛选结果”而不是全部消息，保证用户预期与导出结果一致。
  - 复用现有消息内容提取链路（Markdown/HTML 提取），确保公式与代码块处理策略一致。
  - 导出元数据写入筛选条件（角色筛选、关键词筛选），便于后续追溯导出上下文。
- 取舍说明：
  - 暂不做复杂导出模板定制，优先交付稳定导出能力。
  - 暂不新增设置项，继续复用“默认导出格式”作为快速导出依据。
- 权限/数据/隐私考虑：
  - 全流程本地执行，不新增网络调用与扩展权限。
  - 导出文件由用户主动触发下载，不做后台自动上传。

## 5. 实现
- 主要变更文件：
  - `src/content/conversationExport.ts`
  - `src/content/App.tsx`
  - `README.md`
  - `PRD.md`
- 核心逻辑：
  - 在 `conversationExport.ts` 暴露单节点提取能力：
    - `extractMessageMarkdownFromNode`
    - `extractMessageHtmlFromNode`
  - 在 `App.tsx` 新增时间线导出流程：
    - 从 `filteredTimelineItems` 取当前筛选节点
    - 基于 `timelineNodeMapRef` 回查 DOM 节点并提取内容
    - 生成 Markdown/HTML 导出文本并下载
    - 导出状态通过 `timelineStatus` 反馈
  - 时间线卡片头部新增按钮：
    - 快速导出时间线（按默认导出格式）
    - 时间线 MD
    - 时间线 HTML
- 数据模型/存储变更：
  - 无新增持久化字段。
- 配置/Manifest 变更：
  - 无。

## 6. 验证
- 自动化测试：
  - `npm run typecheck`：通过
  - `npm run build`：通过
- 手工验证步骤：
  1. 打开任意会话详情页，进入“会话时间线”。
  2. 设置角色筛选和关键词筛选，确认列表变化。
  3. 点击“时间线 MD”，检查下载文件包含筛选元信息和节点内容。
  4. 点击“时间线 HTML”，检查下载页面可正常查看代码块和公式渲染。

  - 构建与类型检查通过，时间线节点导出链路可用。

## 7. 风险与回滚
- 已知风险：
  - ChatGPT DOM 结构变化可能影响节点内容提取稳定性。
  - 极长会话在大量节点导出时，浏览器生成文件会有短暂卡顿。
- 回滚策略：
  - 回退 `App.tsx` 时间线导出按钮和回调。
  - 回退 `conversationExport.ts` 新增导出函数。
  - 回退 `README.md` 与 `PRD.md` 的能力说明。
