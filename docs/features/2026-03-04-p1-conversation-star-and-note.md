# FEAT-012：P1 会话星标与备注（自定义元数据）

## 1. 元信息
- 功能 ID：FEAT-012
- 功能名称：实现会话星标、备注与筛选联动
- 日期：2026-03-04
- 作者：Codex
- 分支：N/A（工作区直接更新）

## 2. 概述
参考 `gemini-voyager` 的“重要会话快速定位”思路，在当前会话索引中新增星标与备注能力。用户可为每条会话打星和写备注，并支持“仅星标”筛选，以及把备注纳入统一搜索，提高长对话管理效率。

## 3. 需求映射
- 对应 PRD 章节：6.2（P1 会话级备注与自定义元数据）、8（FR-04 搜索增强）
- 覆盖的用户场景：
  - 用户希望快速标记重点会话并集中查看。
  - 用户希望给会话补充自定义描述，便于后续回查。
- 本次范围：
  - 会话元数据新增 `starred` 与 `note`
  - 列表内星标操作、备注编辑
  - 搜索支持匹配备注，筛选支持“仅星标”
- 本次不包含：
  - 备注富文本
  - 备注历史版本

## 4. 设计
- 关键设计决策：
  - 不新增存储键，直接扩展 `classificationState.metaByConversationId`。
  - 元数据清洗逻辑统一在 store 层处理，避免脏数据写入。
  - 元数据为空（无 folder/tag/star/note）时自动回收，保持存储紧凑。
- 取舍说明：
  - 备注长度限制 240 字，优先保证列表性能与可读性。
- 权限/数据/隐私考虑：
  - 数据仅保存在本地 `chrome.storage.local`。

## 6. 验证
- 自动化测试：
  - `npm run typecheck`
  - `npm run build`
- 手工验证步骤：
  1. 在会话列表点击“☆ 星标/★ 已星标”，刷新页面确认状态保留。
  2. 输入备注后使用关键词搜索，确认可命中备注内容。
  3. 点击“仅星标”，确认仅显示被星标会话。
  4. 与文件夹/标签筛选叠加，确认筛选逻辑正确。
- 测试结果摘要：
  - 类型检查和构建通过，元数据与筛选链路可用。

## 7. 风险与回滚
- 已知风险：
  - 会话条目很多时，备注编辑可能带来更高写入频率。
- 回滚策略：
  - 回退 `classificationStore.ts` 元数据扩展与 `App.tsx` 对应 UI/逻辑。
