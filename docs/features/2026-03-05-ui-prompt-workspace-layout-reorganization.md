# 功能记录

## 1. 元信息
- 功能 ID：UI-016
- 功能名称：提示词工作台排版重组（概览 / 筛选 / 编辑分区）
- 日期：2026-03-05
- 作者：Codex
- 分支：main

## 2. 概述
- 针对“提示词页一坨、排版奇怪”的反馈，重构提示词页布局层级。  
- 将原先松散分布的控件重组为三个清晰区域：概览信息、筛选区、编辑区；统一间距、对齐与控件宽度。

## 3. 需求映射
- 对应 PRD 章节：`FR-05 提示词库`
- 覆盖的用户场景：
  - 用户在窄面板中也能快速理解“状态信息 -> 筛选动作 -> 编辑动作”的顺序。
  - 筛选/批量按钮不再出现拥挤或断裂式排布。
- 范围说明（In scope / Out of scope）：
  - In scope：提示词页结构与样式重排。
  - Out of scope：提示词增删改查与导入导出逻辑。

## 4. 设计
- 关键设计决策：
  - 移除提示词页顶部单独高亮区，改为在同一主卡片中展示概览信息。
  - 新增筛选面板与编辑面板容器，统一边框和内边距节奏。
  - 筛选区使用两列网格布局，按钮与下拉强制同宽；窄屏自动降为单列。
- 取舍说明：
  - 增加少量容器层级，换取信息架构清晰和视觉一致。

## 5. 实现
- 主要变更文件：
  - `src/content/App.tsx`
  - `src/content/index.tsx`
- 核心逻辑：
  - `App.tsx`：提示词页 JSX 重排，新增 `gv-prompt-overview`、`gv-prompt-filter-panel`、`gv-prompt-editor-panel` 等结构。
  - `index.tsx`：新增提示词区块样式，统一网格对齐、间距和响应式规则。
- 数据模型/存储变更：
  - 无。

## 6. 验证
- 自动化测试：
  - `npm run typecheck` 通过。
  - `npm run build` 通过。
- 手工验证步骤：
  1. 打开“提示词库”页面，检查信息顺序是否为概览 -> 筛选 -> 编辑。
  2. 验证筛选区下拉与按钮对齐（同宽、同行）。
  3. 检查编辑区标题/标签/正文输入和底部状态栏间距。
- 测试结果摘要：
  - 提示词页排版更规整，视觉层级更清晰，避免“控件堆叠成坨”的观感。
