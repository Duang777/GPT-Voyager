# 修复：Markdown 数学公式双反斜杠与下划线问题

## 1. 元信息
- 功能 ID：FIX-003
- 功能名称：修复 Markdown 导出中 `\\[...\\]` 定界符和公式命令反斜杠问题
- 日期：2026-03-04
- 作者：Codex
- 分支：N/A（工作区直接更新）

## 2. 概述
针对“HTML 导出正常，但 Markdown 仍有部分公式渲染异常”的反馈，继续修复数学文本归一化流程。重点处理双反斜杠定界符和公式体内命令反斜杠异常，减少 KaTeX/MathJax 渲染报错。

## 3. 需求映射
- 对应 PRD 章节：8（FR-06）、15（DoD）
- 覆盖的用户场景：
  - 用户导出的 Markdown 中数学公式能更稳定渲染。
- 本次范围：
  - 支持识别 `\[` 与 `\\[` 两种数学块开闭定界符
  - 数学体中将 `\\begin` 等命令恢复为 `\begin`
  - 取消高风险的全局下划线转义流程，避免误伤数学表达
- 本次不包含：
  - 所有非标准 LaTeX 文本的自动纠错

## 4. 设计
- 关键设计决策：
  - 数学块提取后做局部归一化，不对全文做激进替换。
  - 保留代码块隔离，避免代码段被误改写。
- 取舍说明：
  - 仍需依赖原始输出文本质量，极端异常公式可能仍需人工调整。
- 权限/数据/隐私考虑：
  - 无新增权限，无新增存储。

## 5. 实现
- 主要变更文件：
  - `src/content/conversationExport.ts`
- 核心逻辑：
  - `normalizeLatexDelimiters` 改为支持 `\\{1,2}` 定界符匹配
  - 新增 `normalizeMathBody` 修复命令双反斜杠与 `\_`
  - 移除高风险 `escapeBareUnderscoreTokens` 流程
- 数据模型/存储变更：
  - 无
- 配置/Manifest 变更：
  - 无

## 6. 验证
- 自动化测试：

- 手工验证步骤：
  1. 导出包含 `\\[ ... \\]` 和 `\\begin{cases}` 的会话
  2. 在支持数学渲染的 Markdown 查看器中打开
  3. 核查公式报错是否明显减少
- 测试结果摘要：
  - 类型检查与构建通过，数学导出兼容性提升。
