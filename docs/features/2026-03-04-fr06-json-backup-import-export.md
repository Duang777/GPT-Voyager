# FEAT-010：FR-06 扩展数据 JSON 备份导入/导出

## 1. 元信息
- 功能 ID：FEAT-010
- 功能名称：实现扩展数据本地 JSON 备份导出与导入恢复（FR-06 补全）
- 日期：2026-03-04
- 作者：Codex
- 分支：N/A（工作区直接更新）

## 2. 概述
在“设置中心”新增本地备份能力，支持将当前扩展数据一键导出为 JSON，并支持导入同版本备份进行恢复。覆盖会话索引、分类、提示词库和用户设置，补全 PRD 中 FR-06 的扩展级导入导出要求。

## 3. 需求映射
- 对应 PRD 章节：6.1（P0）、8（FR-06）、11（本地处理）
- 覆盖的用户场景：
  - 用户迁移浏览器或重装环境后，需要快速恢复历史数据。
  - 用户在批量整理前，需要先做本地快照备份。
- 本次范围：
  - 新增 JSON 备份文件生成
  - 新增 JSON 备份解析与校验
  - 新增设置页导出/导入交互
- 本次不包含：
  - 云端同步
  - 增量备份与多版本管理

## 4. 设计
- 关键设计决策：
  - 备份结构统一为 `schemaVersion: 1`，包含 `exportedAt` 和 `data`。
  - 备份解析时复用各模块 sanitize 逻辑，避免重复校验规则。
  - 导入采用“先解析校验，再整体应用并持久化”，避免半成功状态。
- 取舍说明：
  - 当前仅支持 v1 结构，未来通过 schemaVersion 做迁移扩展。
- 权限/数据/隐私考虑：
  - 所有文件读写都在本地浏览器中完成，不上传外部服务。

## 5. 实现
- 主要变更文件：
  - `src/content/dataBackup.ts`
  - `src/content/App.tsx`
  - `src/content/conversationIndex.ts`
  - `src/content/classificationStore.ts`
  - `src/content/promptLibrary.ts`
  - `src/content/settingsStore.ts`
  - `README.md`
- 核心逻辑：
  - `dataBackup.ts`
    - `createBackupPayload`：组装并清洗备份内容。
    - `parseBackupPayload`：解析 JSON、校验 schema、返回清洗后数据。
    - `createBackupFileName`：生成时间戳文件名。
  - `App.tsx`
    - 设置页新增“导出 JSON 备份 / 导入 JSON 备份”按钮。
    - 导入成功后更新运行时状态并同步写回 `chrome.storage.local`。
- 数据模型/存储变更：
  - 无新增持久化键；复用现有四类存储键。
  - 新增备份文件结构（仅导入导出时使用）。
- 配置/Manifest 变更：
  - 无

## 6. 验证
- 自动化测试：
  - `npm run typecheck`
  - `npm run build`
- 手工验证步骤：
  1. 在设置中心点击“导出 JSON 备份”，确认生成 `.json` 文件。
  2. 修改部分数据（如新增提示词、分类），再执行导入备份。
  3. 验证会话索引、分类、提示词、设置均恢复到备份状态。
  4. 刷新页面，确认恢复数据仍持久化。
- 测试结果摘要：
  - 类型检查和构建通过，导入导出链路可用。
